1. Introduction - Why Link Layer Matters

The link layer (Layer 2) handles node-to-node data transfer between directly connected devices. It transforms raw physical layer bits into structured frames with addressing, error detection, and access controlâ€”enabling your computer to communicate with your router and other local network devices.

The Link Layer operates at Layer 2 of the OSI model

2. Main Content - Core Concepts

Framing and Addressing

The link layer encapsulates packets into frames containing header (source/destination MAC addresses), payload (data), and trailer (error detection).

Ethernet frame structure with MAC addresses and CRC

Error Detection

Physical transmission introduces errors. The link layer detects these using:

Parity Check

Adds a bit to make total 1s even/odd. Detects single-bit errors only.

Checksum

Sums data segments and compares at receiver. Used in TCP/UDP.

Cyclic Redundancy Check (CRC)

Uses polynomial division to generate a checksum. Most powerful method used in Ethernet and Wi-Fi.

Comparison of error detection techniques

Medium Access Control (MAC) Protocols

MAC protocols coordinate channel access to prevent collisions.

Channel Partitioning

TDMA: Time slots for each device. FDMA: Frequency bands for each device.

Random Access

CSMA/CD (Collision Detection)

Used in traditional Ethernet. Detects collisions, stops transmission, retries after random backoff.

CSMA/CA (Collision Avoidance)

Used in Wi-Fi. Uses acknowledgments and RTS/CTS to reserve channel.

CSMA/CD protocol showing collision detection and exponential backoff

Taking Turns

Polling: Master node invites slaves to transmit. Token Passing: Token holder transmits.

3. Link Layer Technologies

Ethernet (802.3)

Wired LAN using switches for full-duplex communication. Speeds: 10 Mbps to 400 Gbps.

Wi-Fi (802.11)

Wireless LAN on 2.4/5/6 GHz. Uses CSMA/CA for channel access.

DOCSIS

Internet over cable TV infrastructure. Uses TDMA upstream, TDM downstream.

Wired Ethernet vs Wireless Wi-Fi architectures

Virtual LANs (VLANs)

VLANs logically segment networks using 802.1Q tags. Devices on different VLANs need a router to communicate, providing security and broadcast isolation.

VLAN configuration showing logical network segmentation

4. Real-World Application

Case Study: Home Network Communication

When browsing a website via Wi-Fi:

  1. Framing: Laptop encapsulates HTTP request with its MAC address (source) and router's MAC (destination)
  2. Medium Access: CSMA/CA checks channel is idle before transmitting
  3. Error Detection: CRC checksum included; router discards if corrupted
  4. Acknowledgment: Router sends ACK; TCP handles retransmission if needed

This process repeats at each hop: laptop â†’ router â†’ ISP â†’ web server.

5. Key Takeaways

  • Link layer provides node-to-node communication using frames with MAC addresses
  • Error detection (parity, checksum, CRC) identifies corrupted framesâ€”upper layers handle retransmission
  • MAC protocols: Channel partitioning (TDMA/FDMA), random access (CSMA/CD, CSMA/CA), taking turns (polling/token)
  • Ethernet uses switches for full-duplex; Wi-Fi uses CSMA/CA for collision avoidance
  • Technologies like Ethernet, Wi-Fi, and DOCSIS implement link layer for different media
  • VLANs provide logical segmentation for security and broadcast control
